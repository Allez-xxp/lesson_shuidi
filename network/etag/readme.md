  有什么问题？
  文件在传输过程中一请求一响应，有必要吗？
  1.浏览器端会缓存的，以便下次更快打开；会清空缓存
  浏览器没必要发送请求，其前提是后端图片没有改变
  2.服务器端的图片不会这么改变，带宽资源（节约），没必要再次去传输
  用户修改了图片，需要通知浏览器端重新下载

- etag是作用于服务器，根据当前文件的内容，生成一个唯一的标识，只要内容不变,就不重传的机制

- 如何缓存文件，省服务器的带宽，客户端直接使用浏览器缓存的文件 etag
1. 服务器端返回文件的同时，在http响应头中带有Etag的值
   值是该文件的唯一标识，hash
2. 浏览器再次请求文件时，request header中带有 If-None-Match:http协议 1.0/1.0-1 2.0 ，If-None-Match:
   浏览器有这个缓存，可以直接用吗？
   询问服务器，图片是否修改？
   没修改，直接使用，缓存
